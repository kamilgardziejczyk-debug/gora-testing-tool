# Guidelines for gora-testing-tool project

## General Rules
1. All code should be written in Python 3.8 or higher.
2. Follow PEP 8 style guide for Python code.
3. Use meaningful variable and function names.
4. Write docstrings for all functions and classes.
5. Use type hints for function parameters and return types.
6. Functions should not exceed 50 lines of code.
7. Avoid global variables; use classes or function parameters to manage state.
8. Handle exceptions gracefully and provide informative error messages.
